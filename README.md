# Storyplay

**Visual novels from your audiobooks.**

Storyplay turns any audiobook + ebook pair into a synced reading experience: word-timed text follows the narration, backgrounds change with the scene, characters appear when they speak, and key moments surface as inline illustrations. Books are alignment data (text + audio + timing). **Packs** layer the visuals on top — characters, backgrounds, and key-moment art — so the same book can look like a graphic novel, a vintage film, a comic, or anything else you want to make.

Open source. MIT-licensed. Ships with public-domain demo books and demo packs so you can run it without any AI generation. The reader is a single HTML page plus a small Python static server — no build step, no framework, no telemetry.

## One-command install

```bash
curl -fsSL https://raw.githubusercontent.com/favstats/storyplay/main/install.sh | bash
```

That clones the repo into `~/storyplay`, fetches the three public-domain demo books from Project Gutenberg, downloads the curated public-domain pack art from Wikimedia, builds the library, and opens it in your browser. The only prerequisites are `git` and Python 3.10+ — both already installed on most Macs and Linux machines.

If you'd rather see what's happening:

```bash
git clone https://github.com/favstats/storyplay.git ~/storyplay
cd ~/storyplay
python3 tools/setup-demos.py      # one-time: fetch demo text + art
./start.sh                         # open library in browser
```

Everything is idempotent — re-running `setup-demos.py` skips books that are already on disk.

## Status

Alpha. The reader, the pack loader, the library, and the public-domain demos work end-to-end as text + curated art. Adding word-by-word audio sync to a book is an optional extra step using [Storyteller](https://gitlab.com/smoores/storyteller) — see [Bring your own book](#bring-your-own-book) below.

## Layout

```
storyplay/
├── index.html        — library landing page
├── reader.html       — the reader itself (loaded with ?book=&pack=)
├── reader.{js,css}
├── books/
│   └── <slug>/
│       ├── book.json       — title, creator, default pack, cover
│       ├── manifest.json   — generated by build.py from aligned EPUB
│       └── content/        — EPUB-aligned text + audio + media overlays
├── packs/
│   └── <slug>/
│       ├── pack.json       — pack metadata, targets which book
│       ├── characters/     — portrait images
│       ├── backgrounds/    — scene backgrounds
│       ├── moments/        — key-moment illustrations
│       └── chapter*.json   — per-chapter scene/speaker/moment map
├── tools/
│   ├── build.py            — EPUB → manifest.json
│   ├── generate-images.py  — optional: AI image gen for personal packs
│   ├── pack-zip.py         — bundle a pack as .spk
│   └── pack-install.py     — install a .spk into packs/
└── docs/
    ├── pack-format.md
    ├── creating-a-pack.md
    └── contributing-art.md
```

## Demo books (public domain)

Ships with three demo books and matching packs that use **public-domain art only** — no AI:

- **Dracula** (Bram Stoker, 1897) — multi-narrator epistolary novel. Pack uses *Nosferatu* (1922) stills and period engravings.
- **Frankenstein** (Mary Shelley, 1818) — gothic, rich speaker cast. Pack uses Theodor von Holst's 1831 frontispiece and 19th-century engravings.
- **A Scandal in Bohemia** (Arthur Conan Doyle, 1891) — a single Sherlock Holmes story. Pack uses Sidney Paget's original *Strand Magazine* illustrations, drawn for the actual scenes.

Audio comes from LibriVox; text comes from Project Gutenberg.

## Bring your own book

Two paths, depending on how much you want.

### Text-only (no audio sync, no Storyteller)

1. Drop an unpacked EPUB into `books/<slug>/content/` so that `books/<slug>/content/package.opf` exists.
2. Write `books/<slug>/book.json` with `slug`, `title`, `creator`, and `license`.
3. `python3 build.py <slug>` writes a manifest. `python3 tools/build-library.py` re-indexes the library.
4. Reload — your book is there. You can build a pack for it whenever you want.

This is the path the demo books use. The reader still renders text and any pack visuals you've authored; only word-by-word audio sync is missing.

### Full audio sync (Storyteller alignment)

Word-by-word audio sync is an optional upgrade. The three demos can be lifted from text-only to fully aligned by running:

```bash
python3 tools/fetch-librivox.py dracula        # downloads LibriVox audio
python3 tools/fetch-librivox.py frankenstein
python3 tools/fetch-librivox.py sherlock-bohemia
```

then feeding `books/<slug>/audio/` + the EPUB to [Storyteller](https://gitlab.com/smoores/storyteller). The full walk-through is in **[docs/audio-alignment.md](docs/audio-alignment.md)**. Plan for 2-24 hours of CPU time per book — it's Whisper transcribing the entire audiobook. You can let it run overnight.

For your own books: any EPUB + any audiobook pair (m4b / mp3 / aac) works the same way.

## Packs

A pack is a folder of images plus chapter JSONs that map them to sentences in a book. It's just a directory; zip it up and you have a `.spk` file that anyone can drop into their `packs/` folder. Packs can be hand-curated (recommended: distinctive look, no AI), AI-generated (`tools/generate-images.py`), or a hybrid.

See **[docs/pack-format.md](docs/pack-format.md)** for the schema and **[docs/creating-a-pack.md](docs/creating-a-pack.md)** for a walkthrough.
