# Storyplay

**Visual novels from your audiobooks.**

Storyplay turns any audiobook + ebook pair into a synced reading experience: word-timed text follows the narration, backgrounds change with the scene, characters appear when they speak, and key moments surface as inline illustrations. Books are alignment data (text + audio + timing). **Packs** layer the visuals on top — characters, backgrounds, and key-moment art — so the same book can look like a graphic novel, a vintage film, a comic, or anything else you want to make.

Open source. MIT-licensed. Ships with public-domain demo books and demo packs so you can run it without any AI generation. The reader is a single HTML page plus a small Python static server — no build step, no framework, no telemetry.

## Status

Alpha. The reader, the pack loader, and the demo packs work. The audio alignment pipeline (Storyteller / Whisper) is a separate dependency that you run once per book.

## Quick start

```bash
cd ~/storyplay
./start.sh         # serves on http://localhost:8000
```

Open the URL and pick a book from the library.

## Layout

```
storyplay/
├── index.html        — library landing page
├── reader.html       — the reader itself (loaded with ?book=&pack=)
├── reader.{js,css}
├── books/
│   └── <slug>/
│       ├── book.json       — title, creator, default pack, cover
│       ├── manifest.json   — generated by build.py from aligned EPUB
│       └── content/        — EPUB-aligned text + audio + media overlays
├── packs/
│   └── <slug>/
│       ├── pack.json       — pack metadata, targets which book
│       ├── characters/     — portrait images
│       ├── backgrounds/    — scene backgrounds
│       ├── moments/        — key-moment illustrations
│       └── chapter*.json   — per-chapter scene/speaker/moment map
├── tools/
│   ├── build.py            — EPUB → manifest.json
│   ├── generate-images.py  — optional: AI image gen for personal packs
│   ├── pack-zip.py         — bundle a pack as .spk
│   └── pack-install.py     — install a .spk into packs/
└── docs/
    ├── pack-format.md
    ├── creating-a-pack.md
    └── contributing-art.md
```

## Demo books (public domain)

Ships with three demo books and matching packs that use **public-domain art only** — no AI:

- **Dracula** (Bram Stoker, 1897) — multi-narrator epistolary novel. Pack uses *Nosferatu* (1922) stills and period engravings.
- **Frankenstein** (Mary Shelley, 1818) — gothic, rich speaker cast. Pack uses Theodor von Holst's 1831 frontispiece and 19th-century engravings.
- **A Scandal in Bohemia** (Arthur Conan Doyle, 1891) — a single Sherlock Holmes story. Pack uses Sidney Paget's original *Strand Magazine* illustrations, drawn for the actual scenes.

Audio comes from LibriVox; text comes from Project Gutenberg.

## Bring your own book

You need:

1. An audiobook (m4b / mp3 / aac)
2. The matching ebook (epub)
3. [Storyteller](https://gitlab.com/smoores/storyteller) (Docker-based) to align them into sentence-level timing data.

Once Storyteller produces the aligned EPUB, point `tools/build.py` at it and you get a manifest the reader can play.

## Packs

A pack is a folder of images plus chapter JSONs that map them to sentences in a book. It's just a directory; zip it up and you have a `.spk` file that anyone can drop into their `packs/` folder. Packs can be hand-curated (recommended: distinctive look, no AI), AI-generated (`tools/generate-images.py`), or a hybrid.

See **[docs/pack-format.md](docs/pack-format.md)** for the schema and **[docs/creating-a-pack.md](docs/creating-a-pack.md)** for a walkthrough.
